Arquitectura del Pipeline

El flujo estructurado de los datos de ventas de la empresa

1. Ingesta y Cálculo Vectorizado

El flujo se inicia cargando los archivos de ventas diarias en formato CSV de forma distribuida. SparkSession se encarga de analizar los tipos de datos subyacentes e inicializar los nodos de cómputo.

Calculamos la columna calculada total_venta mediante aritmética vectorizada, evitando loops tradicionales en Python y maximizando la escalabilidad con volúmenes masivos de registros:

  • Esquema inferido automáticamente para evitar discrepancias de tipos.
  • Casteo numérico explícito (cantidad a int, precio a double).
  • Cálculo en paralelo de: total_venta = cantidad * precio.

Esquema de Procesamiento

# Carga diferida en PySpark
df = spark.read.csv("ventas.csv", header=True, inferSchema=True)

# Cálculo en paralelo
df_resultado = df.withColumn(
    "total_venta", 
    col("cantidad") * col("precio")
)

# Escribir a Parquet de forma atómica
df_resultado.write.mode("overwrite").parquet("ventas_procesadas.parquet")

2. Persistencia Columnar (Parquet)

El almacenamiento en Parquet ofrece una optimización drástica para fines analíticos. A diferencia de CSV que procesa datos fila por fila en disco, Parquet organiza los bytes de forma columnar.

Esto reduce dramáticamente el uso de almacenamiento físico en disco gracias a la compresión nativa y permite a la aplicación de visualización en Streamlit leer grandes históricos de datos casi al instante.

  • Organización física de datos optimizada por columnas.
  • Soporte nativo para lectura rápida mediante pyarrow en Pandas y Streamlit.
  • Metadatos autocontenidos y compresión eficiente (Snappy).

3. Inteligencia Predictiva (Machine Learning)

Para dotar de valor agregado a la analítica de ventas, desarrollamos un cuaderno Jupyter interactivo (modelo_ventas.ipynb) para entrenar y evaluar un modelo de Machine Learning.

Entrenamos un modelo de Random Forest Regressor usando un dataset de entrenamiento representativo generado sintéticamente. El modelo aprende las variaciones de precios y patrones de volumen según el tipo de producto.

  • Algoritmo: Random Forest Regressor de scikit-learn.
  • Features: Producto (One-Hot Encoded), cantidad y precio base.
  • Persistencia: Exportado en modelo_ventas.pkl con joblib.
  • Simulador: Integrado directamente en Streamlit para interactuar y modelar predicciones de ventas dinámicas.

Entrenamiento del Modelo

from sklearn.ensemble import RandomForestRegressor
